NEVER set a target number for rules, sessions, or any count. The source material dictates every number.

**Protocol:**
1. Before encoding a chapter: read it, count predictive verses, count distinct outcomes per verse, count conditional clauses. Write these in the coverage map BEFORE encoding.
2. Before encoding a text: create coverage map with chapter-by-chapter ACTUAL verse counts from the source. Not estimates.
3. During encoding: no session targets. A session ends when a chapter is complete, not when a number is hit.
4. Coverage map has two columns: "verses in source" and "rules encoded." If rules < verses, chapter is incomplete.

**Why:** Anchoring to numbers (130/session, ~550 per text, etc.) caused 15 sessions of truncated rules, required exhaustiveness backfill, and was the root cause of the Phase 1A undercount. Every time a number appears in a plan, it becomes an anchor that constrains depth.

**How to apply:** When creating a coverage map, NEVER write "~130" or "~550." Write the actual verse count from reading the chapter. If you haven't read the chapter, write "TBD — must read before encoding." The number column must come from the text, never from arithmetic.
