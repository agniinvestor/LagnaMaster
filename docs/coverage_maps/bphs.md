## Encoding Protocol

**Source verse count (mandatory before encoding):**
Each "TBD" must be replaced with the actual sloka count from reading the chapter
in Santhanam's translation. The number comes from reading, not estimating.

**Rules per chapter:**
Each predictive verse produces 1+ rules. A verse with multiple outcomes produces
multiple rules. A verse with conditional clauses produces modifiers on the rule.
The rule count is an OUTPUT of reading, not an INPUT to planning.

**Session boundaries:**
A session ends when a chapter is fully encoded. Not when a number is hit.
If Ch.24 (Bhava Lords, 148 slokas) takes 3 sessions, it takes 3 sessions.

**Concordance:**
Every BPHS rule must be cross-referenced against existing Saravali, BVR, and LP
rules during encoding. BPHS is the primary source — its concordance with secondary
texts validates both.
